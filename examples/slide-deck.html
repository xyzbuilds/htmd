Part 03

The problem

Agents are great at writing prose but terrible at writing rich HTML.

03 / 13 htmd — the bidirectional Markdown ↔ HTML bridge for AI agents

Three options for rich agent output

  1. Markdown — cheap to author but visually flat
  2. Hand-written HTML — ~2.5× the tokens, agent skips polish (no dark mode, no accessibility, no print)
  3. htmd — emit ~150 tokens of YAML, get back a polished interactive widget with a Copy-back button
04 / 13 htmd — the bidirectional Markdown ↔ HTML bridge for AI agents Option 3 wins.
Three doors. Door 1 is cheap but ugly. Door 2 is rich but expensive. Door 3 is cheap AND rich AND consistent.

What the agent writes

title: PRs awaiting review

~150 tokens of YAML.

What the human gets back

A real interactive widget with Copy decisions back as a button. Click it; paste it; the agent gets:

1. Decision: Approve
   Reason: "lgtm — ship it"

Part 09

How agents call it

It's a Node CLI. Claude invokes it through Bash.

09 / 13 htmd — the bidirectional Markdown ↔ HTML bridge for AI agents
From the SKILL.md decision tree

Three commands an agent uses

bash 
# Render multiple widgets from one markdown file (the main case)
htmd compose review.md --out review.html

# Suggest which template fits a plain markdown region
htmd detect notes.md --json

# Recover state from an interacted-with HTML file
htmd extract review.html

SKILL.md ships with htmd — Claude loads it and learns when each command applies. No new training, just a skill file.
