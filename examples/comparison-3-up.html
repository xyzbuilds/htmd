How should we deploy our LLM service?

Three approaches for serving Claude in production at our scale (~50 RPS, mostly bursty)

01

Direct API calls

Each request hits Anthropic's API directly. Zero infrastructure on our side.

Pros

  • Zero ops burden — no servers to manage
  • Always on the latest model version
  • Built-in safety, abuse prevention

Cons

  • Per-token cost adds up at scale
  • Latency floor set by network round-trip
  • Rate limits apply across the org

When to use

Prototypes, low-volume products, anything < 1M tokens/day.

Code

const r = await client.messages.create({
  model: "claude-opus-4-7",
  messages: [{role: "user", content: prompt}]
});
02

Self-hosted proxy + cache

Thin proxy in front of Anthropic API with prompt caching, retries, and a local request cache.

Pros

  • Cache hits eliminate cost for repeated prompts
  • Centralized observability and rate limiting
  • Can swap models per route without app changes

Cons

  • Adds a hop and a service to maintain
  • Cache invalidation is hard
  • Still beholden to upstream availability

When to use

Mid-scale apps where you have repeated structured prompts and want centralized control.

Code

app.post('/llm', async (req, res) => {
  const cached = await cache.get(hash(req.body));
  if (cached) return res.json(cached);
  // ...
});
03

Batch + scheduled offline

Queue requests, send them through Anthropic's batch API at off-peak times for 50% discount.

Pros

  • 50% cheaper per token via batch pricing
  • Naturally rate-limit friendly
  • Easy to retry whole batches on failure

Cons

  • Not viable for interactive UX (>1min latency)
  • Requires job queue infrastructure
  • Complex to mix with realtime requests

When to use

Background processing — summarization, classification, content generation pipelines.

Code

const batch = await client.messages.batches.create({
  requests: items.map(toBatchReq)
});
Verdict

Start with Direct API calls while you're prototyping. As soon as you cross ~$1k/mo in API spend or need observability, move to a self-hosted proxy with caching. Use batch for any non-interactive workload — it's free 50% off.
